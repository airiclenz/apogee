- [ ] Auto mode silently degrades to Approval-gating terminal commands on hosts without filesystem confinement (landlock reports `ENOSYS` in most containers/VMs, so `Capabilities().FSWrite == false`). The gating itself is **correct** — ADR 0012's "confine if you can, gate if you can't" (`resolveLadderAuto`, `internal/agent/resolution.go`) — so this is **not** a resolution defect. The defect is that nothing tells the user: `internal/agent/loop.go` refuses Auto only on a *nil* `Confiner`, so a present-but-incapable backend enters Auto with no startup notice, and every terminal command then prompts with no explanation. Needs a startup notice naming the active backend, its capabilities, and the `confine-to-workspace: false` opt-in (the sanctioned "I am the sandbox" path for disposable VMs) — **and an offer to apply that setting in place**, so the user doesn't have to go find `~/.apogee/config.yaml` to get the Auto they asked for. The offer is a real reduction in enforcement, so it carries constraints (truthful blast-radius wording, no default-yes, session-scope offered before persistence, visible/reversible config write, and an open question about global-vs-host scope). → see **TODO.md → "Auto-mode confinement degradation is silent"**.
