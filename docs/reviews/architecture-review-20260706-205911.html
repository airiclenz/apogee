Problem — the Exchange is a first-class CONTEXT.md term with no module: its boundary is re-derived ad-hoc in seven places across three packages, and Exchange-lifetime state (Deferred Response Actions, Mechanism fired-markers) is faked with request-scoped English strings and a Session-scoped queue — the root of all three High findings in the 07-06 review.

Solution — a domain Exchange working value (like LoopView): the conversation owns it, the loop and Mechanisms consume it; deferral scope and fired-marker scope live inside it.

  • Locality: boundary bugs concentrate in one module
  • Leverage: one interface, seven call sites
  • Four deferral-clearing points collapse to one lifecycle
  • Exchange semantics testable directly, not via loop runs
Sequencing + ADR 0014: the in-flight fixes plan (F1–F8) patches each site tactically — run it first; this candidate then gives those patches one home. No ADR conflict: ADR 0014 §3's "re-derive from honest history each Turn" is preserved — the Exchange module is that re-derivation, implemented once.

Problem — the hook interface hands Mechanism authors raw message lists and a dumb Budget value; five Mechanisms hand-roll the same history scans and token arithmetic, each copy subtly different, while the canonical HistoryExceedsAllocation check sits unreachable in context/.

Solution — extend the hook-facing view with shared queries (token estimation, allocation checks, read/written-path scans); Mechanisms delegate instead of re-deriving.

  • Leverage: every future Mechanism inherits the queries
  • Locality: estimation bugs concentrate in one place
  • Tests assert decisions, not scan mechanics
  • Bench campaign about to breed more Mechanisms
Public surface note: the hook interfaces are public via root aliases — additive methods are a minor bump (ADR 0001/0010); adding to an interface type is breaking for external implementers, so the queries belong on LoopView (a struct-backed working value), not on the hook interfaces themselves.

Problem — each built-in tool hand-rolls a JSON-schema string, an args struct, decoding, and error-result shaping; the idiom repeats ~19 times with subtle variation, and ADR 0002 makes tools an open extension point, so third-party authors copy the ritual too.

Solution — a deeper tool-definition module (schema builder, decoder factory, result helpers); built-ins collapse into declarations plus behaviour.

  • Leverage: paved path for tool authors (ADR 0002)
  • Locality: schema/decoding conventions in one place
  • Interface shrinks; implementation absorbs the ritual
  • Less per-tool test boilerplate

Top recommendation

Give the Exchange a home

Three independent signals converge on it: the engine explorer found seven ad-hoc boundary computations, the mechanisms explorer found Exchange-scoped state faked with request-scoped markers, and the 07-06 code review traced all three of its High findings to exactly this gap — and nominated the scoping for this review by name. Sequence it right after the in-flight tactical fixes plan lands: the plan patches the call sites, the Exchange module then concentrates them, and candidate 2 (which needs an Exchange-aware view anyway) follows naturally.
