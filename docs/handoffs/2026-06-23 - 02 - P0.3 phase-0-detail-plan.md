## The task: P0.3 — Phase-0 detail plan

Per TDD §8 P0.3: *"Write the Phase-0 detail plan (task-level, acceptance criteria, version
pins, CI)."* The broad plan's Phase 0 is intentionally coarse; this turns it into a
task-level checklist. Concretely it should pin down:

1. **Dependency version pins** (plan §3a lists the *justified* set — now choose versions):
   Cobra, Bubble Tea + Lipgloss + Bubbles, the official MCP `go-sdk`, `yaml.v3`, and the
   small utils already in apogee-sim's `go.sum` (`shlex`, `ulid`). Each pin is a deliberate,
   noted choice — not an incidental `go get`. **Re-verify MCP go-sdk maturity** (plan flags
   this for Phase 3, but the pin decision starts here).
2. **CI**: cross-compile Win/Mac/Linux; `gofmt`/`go vet`/`go build`/`go vet -race`; build
   the binary. No tests exist yet — CI gates formatting/vetting now, tests as they land.
3. **Task-level acceptance criteria** for the remaining Phase-0 deliverables (plan §4):
   `apogee --help` ✅ done; the `platform/` interface (shell + path + **public `Confiner`
   interface**, stub backends); the **capstone harness** (below); cancellation as a Phase-0
   API primitive (ADR 0007 / plan §6 #24a).

### Then: the Phase-0 *capstone* harness — needs real bodies (sequence after P0.3)
Plan §4 Phase 0 deliverable: *a throwaway in-process harness that constructs an `Agent`,
`Step`s it, snapshots + resumes, and registers an experimental hook.* This **cannot run
against `panic` stubs** — it needs minimal real implementations of the
construct→`Step`→`Snapshot`→`Resume`→`AddExperimental` path. It is the first place the API
surface is exercised for real, and the natural step after P0.2's compile checkpoint. Fold it
into the P0.3 detail plan as the first *implementation* task. (`Step`/quiescent-boundary
semantics: ADR 0007; cancellation through `Step` is a Phase-0 primitive — plan §6 #24a;
`Step()` recovers at the extension boundary — plan §6 #24c.)
