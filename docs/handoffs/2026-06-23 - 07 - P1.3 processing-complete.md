## The finding that fixed the scope (and three calls to ratify)

The plan's "parse **one** tool-call format (the most common **native/JSON** tool-call shape)"
read ambiguously against the TS oracle, which has four strategies (`native` no-op,
`markdown-fenced`, `custom-regex`, `thinking-stripper`) — the *rich* golden vectors live in
`markdown-fenced` + `thinking-stripper`, but the parenthetical says native. **Ground truth
resolved it:** `apogee-sim` (the bench — the P1.7 deliverable) is built entirely on native
structured `tool_calls`, forcing JSON via grammar-constrained decoding when a server lacks
native support (`apogee-sim/CLAUDE.md`, `apogee_test.go`). So native is literal **and
required** — markdown-fenced would leave P1.7 unable to parse the bench's calls. Confirmed by
the provider's own comments (`internal/provider/wire.go`): `FunctionCall.Arguments` is kept
"verbatim (unparsed) so **processing/ owns parsing**." Calls a reviewer should ratify/reject:

1. **The "one format" is native/JSON, not markdown-fenced.** The provider already extracts the
   wire tool-call shape; processing parses the args string → `domain.ToolCall` and strips
   thinking. markdown-fenced/custom-regex + the full harmony channel set are Phase 3 (per plan
   §6). **Main forward-looking call.**
2. **`processing` depends only on `domain`, not `provider`.** It defines its own
   `NativeToolCall{ID,Name,Arguments}`; the loop adapts `provider.ToolCall → NativeToolCall`
   at the seam (P1.2), honouring "wire types stay provider-local" (ADR 0010). The alternative
   (import `provider`) was rejected to keep the seam clean. Reversible.
3. **Malformed batch fails atomically** (no partial calls reach dispatch) and empty args →
   `{}`. Simplest safe contract; P1.2 owns the eventual dispatch/surface policy and may switch
   to per-call resilience using the same primitive.
