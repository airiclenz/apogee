## What landed (depends only on `internal/{domain,provider,processing,tools}` + stdlib; **zero new module deps**)

**The provider seam is now streaming** (`internal/provider/responder.go`): `Responder` is
`Stream(ctx, Request) iter.Seq[Delta]` — the loop consumes the SSE stream the real `Client`
already emitted (P1.1). `Respond` stays a concrete `Client` method (model discovery, simple
calls) outside the interface, so a fake answers one method. `var _ Responder = (*Client)(nil)`
still holds.

**The engine adopted `domain.Conversation`** as its conversation storage
(`internal/agent/conversation.go`), replacing the throwaway P0.6 `message`/`conversation` pair.
It already carries everything the full machine needs — role-tagged messages with tool calls +
tool-call IDs (the multi-Turn tool loop), the FIFO deferred-action queue (the `ActionDefer`
feed-forward), and a JSON round-trip — so the loop appends to it and `Snapshot` serializes it
directly. Added one method: `domain.Conversation.Append` (Insert-at-Len with a name that reads
at the call site). **This is a deliberate deviation from handoff-09's staging** (which slated
the replacement for P1.6): P1.2 genuinely needs the rich message schema *now*, and adopting the
real, library-complete type here avoids building a parallel throwaway in `agent` that P1.6 would
just delete — strictly less churn and the long-term-correct shape (per the owner's
"optimize for the long term, not lowest churn" directive). **P1.6's remaining scope is the
Session *envelope*** around this payload: serialize `turnIndex`, populate per-message `Extra`
wire fields, document the v1 schema.

**The full Turn/Step state machine** (`internal/agent/loop.go` + new `dispatch.go` + new
`hookrun.go`). One `Step` is one Turn (ADR 0007):

1. consume queued input → **history-rewrite hooks** edit `conv` →
2. **build request** (drain the deferred corrections + role-safe `InjectContext` each + tool
   menu + budget) → **pre-request hooks** shape it (P1.5) →
3. **stream** the reply (`streamResponse` emits a `TokenEvent` per content chunk; accumulates
   text/reasoning/tool calls; the SSE body closes at the terminal Delta) →
4. **parse** tool calls (`processing.ParseNativeToolCalls` at the seam; a malformed call
   degrades to a parse-error path, not a Turn failure) →
5. **post-response hooks** (`ActionRetry` re-calls the Upstream in place, bounded by
   `maxPostResponseRetries`; `ActionIntercept` is read back via `Text()`/`ToolCalls()`;
   `ActionDefer` → `conv.Defer`) →
6. **no tool calls** → commit the assistant message, emit `MessageEvent`,
   `StatusExchangeComplete`. **Tool calls** → commit the assistant tool-call message, then
   **dispatch** each through pre-tool-exec hooks → **Approval** → execute → post-tool-result
   hooks → append the tool message + emit `ToolResultEvent`; `StatusTurnComplete` (the next Step
   calls the Upstream again with the results in context).

`Run` (was the last `panic("sketch")`) steps until the Exchange ends or cancels.

**The two open control-flow calls are settled here:**
- **§6 #6 (streaming + Approval interleave) — stream-then-gate.** The stream is consumed to its
  terminal Delta and the SSE body **closed** before any tool dispatch; Approval is consulted
  synchronously *afterward*, so a blocking `Approver` never holds an open Upstream connection.
  The EventSink sees, per Turn: `TokenEvent`s → [stream ends] → `ToolCallEvent` → `ApprovalEvent`
  (around the blocking `Approve`) → `ToolResultEvent`.
- **§6 #3 (event delivery) — synchronous, in-order, no buffer/drop.** The loop emits
  synchronously in Turn order; the non-blocking contract is the host's (the `EventSink` doc
  states it). The bench consumes Events as Go values in order; a buffered channel adapter for
  the Phase-2 TUI sits behind the same interface.

**Approval gating** (`dispatch.go`): Plan filters the menu to read-only tools and refuses a
write defensively; Ask-Before gates every write (read-only tools skip); Auto gates only
external-effect tools (ADR 0004); `allow-for-session` is cached on the Agent. A nil Approver
when a gate is required refuses (emits an `ErrorEvent`) rather than running unapproved.

**`Config.WorkspaceDir`** (new, `internal/domain/config.go`): when `Config.Tools` is nil and
`WorkspaceDir` is set, `newAgent` resolves the built-in registry via
`tools.NewDefaultRegistry(root)` (`internal/agent` → `internal/tools`, no cycle). Empty both ⇒
tool-less.

**Cancellation & recovery** (preserved across the richer flow):
- **Cancel mid-stream or mid-tool** → `cancelTurn` rolls `conv` back to the boundary the Turn
  began at (drops this Turn's assistant message + tool results), re-queues the drained deferred
  corrections, returns `StatusCancelled` **without** advancing `turnIndex` — the snapshot
  resumes and re-attempts. The user message is kept (input not lost).
- **Tool/hook panic** → recovered at the extension boundary into an `ErrorEvent`; a tool panic
  yields an error tool-result and the loop continues; a pre-request/history panic abandons the
  Turn cleanly. `restoreDeferred` re-queues drained corrections on *every* non-commit path, so a
  best-effort correction is consumed only when a request is actually sent and committed.

## The judgment calls to ratify (four)

1. **Adopt `domain.Conversation` as engine storage now (P1.2), not P1.6.** Deviates from
   handoff-09's staging; justified above (the machine needs rich messages now; adopting the real
   type avoids throwaway duplication; long-term-correct). **Main call.** What's left for P1.6 is
   the Session envelope (turnIndex serialization, Extra wire-field population, documented v1
   schema) — still substantial.
2. **`Responder` seam is streaming-only (`Stream`).** The loop only needs streaming; a minimal
   interface keeps a fake to one method. `Respond` survives as a concrete `Client` method (tested,
   useful). Reversible.
3. **§6 #6 = stream fully, *then* gate** (not Approval mid-stream). The SSE body closes before the
   blocking `Approver` is consulted — no held-open connection, no partial state. The clean,
   robust resolution; documented in `dispatchTools`/`streamResponse` and TDD §6 #6.
4. **Cancellation abandons the whole Turn (rollback), uniformly for mid-stream and mid-tool.**
   Resume re-attempts the Turn from a serializable boundary. A re-run write tool overwrites
   idempotently (ADR 0008); acceptable for Phase 1. The user message is kept across a cancel.

## Next: P1.6 (concrete Session schema + versioning)

`Snapshot` currently serializes only the `domain.Conversation` payload (`encodeConversation`).
**P1.6** wraps it in a versioned envelope that also serializes the loop counter (`turnIndex`) so
resume *continues* at the right Turn rather than restarting at 0 (a documented gap — a resumed
Agent re-zeroes `turnIndex` today), populates per-message `Extra` wire fields (the provider→domain
decode that fills `Message.extra` — load-bearing for round-tripping `reasoning_content`/
`tool_choice`), keeps `Session.Version` rejection, and documents the v1 schema.
**Acceptance:** snapshot → resume → `Step` continues at the correct `turnIndex`; a round-trip
preserves tool-call/result pairing (already true via `domain.Conversation`) and `Extra` fields;
an unknown future `Version` → `ErrSessionVersion`. Then **P1.7** points apogee-sim at the API
(`go.mod replace`, construct/`Step`/score a file-edit task) — the Phase-1 deliverable.
