**Error contract** (the seam P1.2 builds on): an *expected* failure (bad args, missing file,
path escape, oversize) is a `ToolResult{IsError:true}` the model sees and reacts to; the Go
`error` return is reserved for **ctx cancellation** (so P1.2 can map it to `StatusCancelled`).
A panic is **not** caught inside the tool — it surfaces at the loop's extension boundary as an
`ErrorEvent` (ADR 0007), which is P1.2's job, not the tool's.

**Loop wiring is deliberately deferred to P1.2** — exactly as P1.3 (parser) and P1.1 (`Stream`)
landed tested libraries first. `loop.go` / `agent.go` are untouched; no dispatch/executor/
approval here.

## The judgment calls to ratify (four)

1. **Loop wiring → P1.2.** P1.4 is a tested library; dispatch/executor/Approval-gating — incl.
   the §4 acceptance line "a write goes through Approval in Ask-Before" — belong to **P1.2**,
   which the plan's dependency table already assigns ("P1.2 … dispatch tools (P1.4) through
   Approval"). The read/write signal P1.2 needs is `domain.IsReadOnly(tool)`. **Main call.**
2. **Tools take a sandbox root at construction**, not a new `domain.Config` field — direct
   parity with the TS oracle's `workspaceRoot`, keeps `internal/tools` decoupled from `Config`.
   **Forward dependency for P1.2/P1.7:** to call `NewDefaultRegistry(root)` they need a
   workspace root; `domain.Config` has `LibraryDir`/`SessionsDir`/`ConfigDir` but **no
   workspace/working-dir field**. Add `Config.WorkspaceDir` in the task that first wires the
   default tool set (likely P1.2). Reversible.
3. **`grep` defaults to *all* files** (binary + oversize skipped; `include` narrows), **not**
   the oracle's hardcoded TS/JS extension all-list — that default would miss `.go` files in a
   Go repo. Deliberate generalization for a polyglot agent; documented in `grep.go`.
4. **`ReadOnlyTool` is an optional interface** (mirrors the existing `ExternalEffectTool`
   pattern), not a method added to the core `Tool` interface — so existing/embedder tools keep
   compiling and an undeclared tool defaults to **write-capable** (the safe, gating default).
   If P1.2 finds it wants read/write to be mandatory, promoting it to `Tool` is a one-line
   interface change + a build-break that names every offender.
