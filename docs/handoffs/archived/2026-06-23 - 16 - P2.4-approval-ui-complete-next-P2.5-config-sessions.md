## What P2.4 built that P2.5 builds on (all in `internal/tui/model.go`, ADR-0010 grep stays empty)

- **`handleApprovalKey` + the `approvalKeys` map** — `a`→`ApprovalAllow`, `d`→`ApprovalDeny`,
  `s`→`ApprovalAllowForSession`, sent **non-blocking** on the cap-1 `m.pending.Reply`, then
  `m.pending` cleared and state→`running`, **re-arming `m.spinner.Tick`** (the tick chain dies
  while the prompt is up, so any return to `running` must re-arm it — a pattern P2.5 should copy
  if it adds another running-resume path). The decision branch sits **before** the viewport-scroll
  fall-through in `handleKey`; a non-decision key still scrolls the transcript.
- **`approvalPrompt` + `View`** — the prompt block reuses **`prettyJSON`** for the arguments (the
  same formatter as the tool-call entries). `View` **shrinks the viewport on its local value copy**
  (`m.viewport.SetHeight(...)` — value receiver, so the stored layout is untouched) so an extra
  stacked block never pushes the status/input/hint off-screen. If P2.5/P2.6 stacks another
  transient block (e.g. a save-confirm or resume picker), copy this shrink-the-viewport-locally
  trick rather than growing the layout budget.
- **Cancel-while-pending is proven** (`esc` → `busy()` → `stopWorker` → `cancelledMsg` →
  `finishWorker` clears `m.pending`); P2.5's quit path runs through the same `quit()`/`finishWorker`.

## Next: P2.5 — config & session glue (plan §4 P2.5; ADR 0001 roots; §6.1 sessions)

Two pieces, both meant to stay **thin**:

1. **Config precedence (flag > env > file > default), in `cmd/apogee`.** This lives in the
   composition root (`root.go`/`wire.go`), **not** in `internal/tui` — keep the TUI a renderer.
   The state roots are **already resolved** in `wire.go` (`roots.config/library/sessions` →
   `domain.Config.{ConfigDir,LibraryDir,SessionsDir}`) under the single **`~/.apogee`** dotdir
   (owner decision — uniform on every OS, *not* XDG/`os.UserConfigDir`; `--config` overrides; the
   roots are **paths only — no dir creation** yet). The optional `.apogee/config.yaml` /
   `~/.apogee/config.yaml` loader (`yaml.v3`) is added **only if it earns its place** this phase;
   else defer it, leaving `yaml.v3` a pinned-but-unused decision. Don't add a dep for ~30 lines of
   flag plumbing. **Acceptance:** precedence is **table-tested** (flag > env > file > default).

2. **Snapshot-on-quit + `--resume`.** `--resume <file>` is **already wired** (P2.0, riding
   `apogee.Resume`, with round-trip + future-version `ErrSessionVersion` tests) — leave it. What
   P2.5 adds is the **save**: on a clean quit, snapshot the conversation to `SessionsDir`.
   - The seam is ready: **`Engine.Snapshot() (domain.Session, error)`** is already in the narrow
     interface (`tui.go`), and `fakeEngine` in the tui tests already satisfies it. Snapshot/Resume
     are valid **only at a quiescent boundary** (between Steps) — quit at `idle` is such a boundary;
     decide what to do if the user quits mid-`running` (snapshot the last boundary, or skip).
   - **Gap to close:** the TUI has **no `SessionsDir`** today. `Options` carries
     `Model/Endpoint/Mode/Bypass/Workspace` only. P2.5 must thread the sessions path (and a way to
     **write** a `domain.Session`) into the TUI. The TUI **may import `internal/session`** to
     encode/write — the ADR-0010 grep is the *root* path `"github.com/airiclenz/apogee"` (exact,
     closing quote), which `internal/...` subpackages do **not** match (that is how the TUI already
     imports `internal/domain`). Cleanest options: add `SessionsDir string` to `Options` and have
     the model call `eng.Snapshot()` then write via `internal/session`; **or** pass a small
     `func(domain.Session) error` saver seam from the composition root (keeps the TUI from owning
     the path/format — fits the "TUI is a renderer" line and the long-term-architecture directive).
     Pick the saver-seam if unsure; it keeps file I/O in the binary.
   - **Acceptance:** a snapshot written by the TUI **round-trips through `--resume`** and
     **continues** the conversation at the right Turn (reusing P1.6's `turnIndex` continuation); a
     `--resume` of a future-version file surfaces `ErrSessionVersion` as a friendly error, not a
     panic (already covered at the wire layer — keep it green). **No session browser UI** (§6).

Then **P2.6** (hermetic `httptest` e2e through the seam + the live-model run). The seam + skeleton
+ fold + approval are all there for it.
