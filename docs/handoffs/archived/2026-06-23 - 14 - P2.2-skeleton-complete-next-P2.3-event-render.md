## What P2.2 built (the skeleton P2.3/P2.4 enrich)

All in `internal/tui`, still driven only by the narrow `Engine` interface (ADR-0010 grep stays empty):

- **`model.go`** — the `Model` (a **value type** with value-receiver `Init`/`Update`/`View`, copied
  every Update per Bubble Tea idiom). Fields: the narrow `Engine`, the display `Options`, the
  program `parent` ctx (the worker derives from it — C4), the Bubbles `textarea` (input) +
  `viewport` (transcript) + `spinner`, the four-state `state ∈ {idle, running, awaitingApproval,
  errored}`, the worker's `cancel CancelFunc`, the `*approvalReqMsg pending`, and `width/height`.
  - **`Update` folds exactly:** the five seam Msgs (`eventMsg`/`approvalReqMsg`/`exchangeDoneMsg`/
    `cancelledMsg`/`errMsg`), `tea.KeyPressMsg`, `tea.WindowSizeMsg`, and `spinner.TickMsg` — nothing
    else, **no agent logic** (C5).
  - **Keys:** `enter`@idle → `submit()` launches the worker via `startExchange` and stores its
    `CancelFunc`; **`enter` while running/awaiting = no-op** (the single-worker invariant); `esc`/
    `ctrl+c` cancel an in-flight worker (`stopWorker`) or quit at idle; `enter`@errored dismisses to
    idle. `submit` ignores blank input and `Reset()`s the box.
  - **`View`** stacks `viewport · status · input · hint` via `lipgloss.JoinVertical`, on the
    **alt-screen** (`v.AltScreen = true` — a v2 *view field*, not a program option). Pre-first-
    `WindowSizeMsg` it shows a placeholder, never a panic.
- **`transcript.go`** — the **C6 entry model**: `[]entry` (typed: user/assistant/toolCall/
  toolResult/error/note, with `depth` for Phase-3 sub-agents) + an in-progress assistant **token
  buffer** (`strings.Builder` + `streaming bool`). `apply(domain.Event)` is an **exhaustive switch
  over all 8 variants**; P2.2 folds **TokenEvent** (→ buffer), **MessageEvent** (→ commit, its text
  canonical), and **ErrorEvent** (→ notice), and records the Turn index in every case. The other
  five cases (`StreamReset`/`ToolCall`/`ToolResult`/`Approval`/`MechanismFired`) are **marked
  `P2.3` stubs on the stable structure** — P2.3 fills the bodies, it does not restructure.

`tui.go`'s `Run` is now real: `tea.NewProgram(newModel(ctx, eng, opts), tea.WithContext(ctx))`,
then **`br.Bind(program)` *before* `program.Run()`** (the late-bound sink/approver reach the live
program the instant the first worker emits). `*tea.Program` satisfies `programSender` exactly as
P2.1 verified.

   `Update` returns the **concrete** model type (`m.input, cmd = m.input.Update(msg)`), `View()`
   returns a **string**. Tests force determinism by **stripping ANSI** (a CSI regexp in
   `model_test.go`) — no golden-file dep was added (keeps the module graph lean).

## Next: P2.3 (the event fold) + P2.4 (the Approval UI)

Both build on this skeleton without touching the threading model — they only fill `transcript.apply`
bodies and add keypress handling:

- **P2.3 (plan §4 P2.3; §0 event-sequence rule):** fill the five stubbed `apply` cases — append
  `ToolCallEvent`/`ToolResultEvent` entries (finalise the pre-tool narration on the **first
  ToolCall of a Turn**), **discard the token buffer on `StreamResetEvent`** (the `discardPending`
  the comment names), record `ApprovalEvent` observationally, and gate `MechanismFiredEvent` behind
  a debug view. Reconcile streamed tokens vs `MessageEvent.Text`. Honour `entry.depth` (indent
  `Depth > 0`). **Acceptance:** golden-ish transcript from a recorded `coreagent`-shaped sequence;
  the StreamReset discard; an inline `ErrorEvent` that keeps the transcript going.
- **P2.4 (plan §4 P2.4; ADR 0004):** when `awaitingApproval`, render `m.pending.Request` (tool,
  args, `Reason`) and handle `a`/`d`/`s` → send the decision over **`m.pending.Reply`** (the
  buffered chan from C3) and return to `running`; a stop key while pending already cancels (the
  `cancelledMsg` path clears `pending`). **Acceptance:** each key yields the right
  `ApprovalDecision` on the reply chan (table test, `-race`); cancel-clears-prompt.

Then **P2.5** (config + sessions / snapshot-on-quit) and **P2.6** (hermetic httptest e2e + the
live-model run). The seam + skeleton are there for all of them.
