## Two judgment calls a reviewer should ratify or reject

1. **Conversation (de)serialization stayed in `internal/agent`, not `internal/domain`.**
   P1.0a's literal text says move it to domain; ADR 0010 layer-2 says "conversation state
   = engine". They conflict. Chose engine because P1.6's real Session payload grows to
   carry engine-only `turnIndex` + deferred actions, so the payload schema belongs with
   the engine; **domain owns only the Session envelope + versioning.** All P1.0 acceptance
   passes either way. Reversible if the owner prefers the literal reading.
2. **`eventBase` → exported `domain.EventBase`, deliberately NOT root-aliased.** The engine
   must construct events (set `Turn`); an unexported base forbids that. The seal
   (`eventDepth()` unexported in `domain`, `EventBase` never re-exported at root) still
   prevents external code from forging new `Event` variants. Verified.

## Next: P1.1 — the real OpenAI-compatible provider

Plan: [`phase-1-detail-plan.md`](../plans/phase-1-detail-plan.md) §4 "P1.1". In short:
`internal/provider` gains an OpenAI-compatible chat-completions client implementing
`provider.Responder` (**non-streaming first, then streaming**), `/v1/models` discovery,
bounded retries + timeouts, and a server-process manager. **Replace `provider.Placeholder`
in place** (engine's `New`/`Resume` bind it in `internal/agent/agent.go`). Add an
`httptest.Server` test so the wire path is hermetic (no live Upstream in CI). Wire types
stay provider-local; the loop translates at `buildUpstreamRequest` (`internal/agent/loop.go`).
TS `openai-compatible-provider` / `model-discovery` / `server-process-manager` are the
behavioural oracle. **Acceptance:** table-driven `httptest` covering request-shape +
response-parse + a retryable 5xx + a timeout; non-streaming and streaming both round-trip;
**zero new module deps** (`net/http`).

Note: P1.2 (the full Turn/Step state machine) is the convergence point and depends on
P1.1+P1.3+P1.4+P1.5. P1.1 can land independently against the existing single-Turn loop.
