## The judgment calls to ratify (five)

1. **Scope = pre-request wiring + tested libraries; post-response/history-rewrite loop
   integration → P1.2.** Identical to how P1.1/P1.3/P1.4 landed tested libraries and deferred
   loop wiring to the convergence. The handoff-08 framing of P1.5 named exactly this ("real
   bodies … **and** wire pre-request mutations"). The plan's second acceptance clause — "a
   post-response `ActionDefer` survives a snapshot/resume boundary and injects on the next
   request" — is delivered **at the primitive level** (Conversation's deferred queue + JSON
   round-trip + `InjectContext`, unit-tested end-to-end in `TestConversationDeferSurvivesRoundTrip`)
   because the **end-to-end loop demonstration needs P1.2** (run post-response hooks) **and P1.6**
   (the unified, serialized conversation). P1.5 depends only on P1.0, so it cannot lean on either.
   **Main call.**
2. **`domain.Request` owns its storage; the engine builds via `NewRequest` and drains via
   `State() RequestState`.** A hook mutates the opaque struct through its method surface; the
   engine (a different package) can't read unexported fields, so the drain is a single, documented
   engine seam — not per-field getters that would pollute the hook namespace. Reversible.
3. **`domain.Conversation` is NOT yet the engine's conversation storage.** The engine still
   snapshots its throwaway `agent.conversation{Messages}`; `domain.Conversation` is a
   library-complete, serializable value that **P1.6 adopts** ("Replace the throwaway
   `conversation{Messages}`"). Unifying them now would pull P1.6 forward. So the deferred-action
   queue + Message `extra` field exist and are tested, but the engine doesn't drive them until
   P1.2/P1.6. **Forward dependency for P1.6.**
4. **Tool menu + sampling project onto `provider.Request` now; `SetExtra` does not.** The menu
   (from `cfg.Tools`, nil-safe) and sampling drain through so `SetTools`/`SetSampling` are real
   end-to-end. `provider.Request` has **no carrier for arbitrary extras** (response_format is a
   Phase-4 grammar concern), so `SetExtra`/`Extra` are real at the domain level but not wired to
   the wire — documented in `toProviderRequest`. Reversible.
5. **Trivial Budget for Phase 1.** `buildRequest` reports `Budget{ContextLimit:
   cfg.Context.MaxContextTokens, CharsPerToken: 4.0}`, `Used: 0` — no token accounting yet (TDD
   §8 #8, explicitly a Phase-1 non-goal). `LoopView.Fired` returns 0 (self-regulation is Phase 4;
   the `fired` map is nil). Both are honest Phase-1 stubs behind real method bodies.

## Next: P1.2 (the convergence)

**P1.2** consumes `provider.Client.Stream` (P1.1) + `processing` (P1.3) + tools/registry (P1.4) +
hook-mutation (P1.5): runs the full state machine, implements `Run` (still `panic("sketch")` in
`agent.go`), and settles §6 #6 (streaming+Approval interleave) and §6 #3 (event backpressure).
When P1.2 wires the rest it will:

- **Tools:** add `Config.WorkspaceDir` (handoff-08 call 2 — `domain.Config` has no working-dir
  field), build the registry via `tools.NewDefaultRegistry(root)`, gate writes via
  `domain.IsReadOnly` + the `Approver`, filter the Plan-mode menu to read-only tools, recover
  tool panics → `ErrorEvent` at the boundary.
- **Post-response hooks:** build a `domain.Response` via `domain.NewResponse(text, thinking,
  calls, finish, req.View())` after parsing, run the post-response hooks, honour the
  `PostResponseDecision` (`ActionRetry` re-calls; `ActionIntercept` reads back `Text()`/
  `ToolCalls()`; **`ActionDefer` → `conv.Defer(decision.Inject)`**).
- **Feed-forward:** at the top of `buildRequest`, drain `conv.TakeDeferred()` and `InjectContext`
  each — this is the loop half of the ActionDefer path P1.5 proved at the primitive level.
- **History-rewrite hooks:** run `RewriteHistory(ctx, conv)` over the conversation.
- **Pre-tool-exec / post-tool-result hooks:** pass `req.View()`/a `LoopView` + the originating
  `ToolCall` (the design's signatures already carry these).

**P1.6** then replaces `agent.conversation` with `domain.Conversation` (deferred actions +
`turnIndex` + preserved Message `Extra` all serialize), and **P1.7** points apogee-sim at the API.
