## 2. Decide this FIRST (TDD ¬ß6 #7) ‚Äî it shapes every Phase-1 body

P0.6 surfaced a real architectural tension and deliberately left it for Phase 1:

- The committed dependency direction is **internal‚Üíroot** (`internal/platform` imports the
  root `apogee` package for `Confiner`; public types live in root). That makes it
  **impossible for the root facade to import `internal/agent` without a cycle** ‚Äî so P0.6 put
  the loop bodies in the **root package** and kept only the root-type-free `Responder` seam in
  `internal/agent`. This **conflicts with the ¬ß3 layout** (`internal/agent/{loop,subagent,modes}`).
- **The call (TDD ¬ß6 #7):** (a) keep the loop in root, `internal/agent` holds only
  provider-shaped seams; (b) add a shared `internal/core` types package both import; or
  (c) flip to apogee-sim's *root‚Üíinternal* aliasing (re-export internal types from root ‚Äî
  also moves `Confiner`, reopens ¬ß6.1). Lowest-churn is (a)/(b); (c) is a larger reshape.
- **Why it must come first:** the provider client, the real loop, and `internal/agent` all
  depend on which way this resolves. Recommend resolving it as a short design note / ADR
  before any Phase-1 body lands.

## 3. Phase-1 worklist (from the plan ‚Äî not re-derived here)

Order and detail are in the broad plan ¬ß4 Phase 1 and TDD ¬ß8 P1 (#4‚Äì8). The headline items:

- **Real provider** ‚Äî OpenAI-compatible HTTP client implementing `internal/agent.Responder`
  (the seam P0.6 already defined); model discovery; retries/timeouts; server-process mgr.
  An `httptest.Server` upgrade of the harness then exercises the wire path.
- **Full Turn/Step state machine** ‚Äî how a `Step` interleaves stream ‚Üí parse ‚Üí hooks ‚Üí tool
  dispatch ‚Üí approval ‚Üí boundary (TDD ¬ß6 #6 streaming+approval interleave is a sub-decision).
- **Concrete Session schema + versioning** (TDD ¬ß8 #7) ‚Äî what serializes into `Session.State`
  beyond the throwaway `conversation`; restore loop counters (`turnIndex`) too ‚Äî P0.6 does
  **not** (documented simplification).
- **Hook-mutation API** (TDD ¬ß6.2, the biggest *public* gap) ‚Äî wire `Request`/`Response`/
  `Conversation` real bodies; P0.6 fires pre-request hooks but does **not** flow their
  mutations into the Upstream request.
- **Minimal tools** + point **apogee-sim** at the Go API (the bench drives the real loop).

**Throwaway P0.6 internals Phase 1 replaces:** `placeholderResponder` (errors until the real
client lands), the minimal `conversation` value, the cycle-check-only registry (no total
order yet ‚Äî ADR 0003 topo-sort is Phase 4).
